My AddOns for example use “Scripting Define Symbols” a lot, in order to detect if another AddOn is installed or not.

Sometimes, this causes a lockup as Unity is looking for such a “Scripting Define Symbol” before the AddOn had a chance to add it (thats usually a red error in the console log). In that case, its always a good idea to clear your Scripting Define Symbols.

After this has been done, most of the weird errors usually dis-appear.

  • Go to your Player Settings via the Menu:
    Edit -> Project Settings -> Player
  • Look for the “Scripting Define Symbols” section in your Player Settings.
  • Select all the text, delete it. And afterwards press Return, Unity will now rebuild all Scripting Define Symbols.

]]> https://indie-mmo.net/knowledge-base/scripting-define-symbols/feed/ 0 Common Mistakes & Problems https://indie-mmo.net/knowledge-base/common-problems/ https://indie-mmo.net/knowledge-base/common-problems/#respond Thu, 31 Oct 2019 14:13:26 +0000 https://indie-mmo.net/?post_type=epkb_post_type_1&p=5637 Scripting Define Symbols

Sometimes, when importing AddOns there are red errors in the console, even though there are no errors in the script. This could be because of Scripting Define Symbols not refreshed properly. Check my other post on how to solve that, it will save you from a lot of trouble! 

Importing/Removing and Re-Importing AddOns

You have to remember: When you add an AddOn, the Scripting Define Symbols will be added to your project. Now, when you remove that AddOn, the Scripting Define Symbols stay in there – but the AddOn is not available anymore. This will also cause errors! Again: You should clear the Scripting Define Symbols (as explained above) when removing AddOns.

Database Lock / No such Column

The database tables/columns change from time to time when I add new functionality to my AddOns. Sometimes this throws a “no such column” error in the console or the database ends up locked. In that case you should delete your database file as well as the journal file if there is one. Both are located in your project folder.

Please note that this will delete all your player data in a live project as well. In that case you have to add/update your database manually.

No Text in UI of UCE AddOns

Several of my AddOns feature editable text colors. If you wonder that no text shows up at all, take a look at the corresponding UI element in your canvas. When exporting a package using Unity, the colors do not export correctly and appear as 100% transparent. Set transparency manually to make your text show up correctly, sadly there is not much I can do about this.

Network Spawning Related Errors

Many network errors happen when you forget to add game objects to your Network Manager registered spawnable prefabs. You must add all objects that can be spawned by any means to that list (this includes visual skill effects), otherwise you will get errors sooner or later. Check the inspector of your NetworkManagerMMO and add all spawnable game objects to it.

Creating New Objects

From my personal experience: Whenever I try to create a competely new Skill or Item, it fails. Therefore I always duplicate an existing Item or Skill and then change the properties.

This is even more important to prefabs like Monsters, Pets and Players. Whenever I try to create one from scratch, it fails. This results in very strange bugs that are hard to track down. Why does this happen? Check the Inspector of a Player for example, see how many components there are and how many settings? Its so complex that it is very easy to overlook something.

Always duplicate one of the default objects like the Skeleton, the Warrior, the Baby Skeleton or any of the Items and/or Skills and use it as your new template !

The Resources Folder

Make sure to always have just one Resources folder in your project and never have any duplicates of the same name. This is especially important with my AddOns as most come with new Items and Skills: You have to move them to your main Resources folder, while not keeping the original ones.

uMMORPG uses a internal dictionary to keep track of all Items and Skills. It scans the main resources folder for those objects. If you don’t keep all your objects in the main Resources folder, that scan will fail and the objects will not be in the Dictionary. This will sooner or later cause errors in your project.

Try on a Blank Project First

I always recommend that you maintain at least 2 unity projects: Your main game project and a secondary project that only contains uMMORPG as well as my AddOns. Try adding new AddOns to the blank project first and see if you can get everything to work correctly. Then, you can add it to your main game project.

This is also very important for error reports: Lots of errors arise if you combine my AddOns with too many other AddOns or Unity Assets that affect how things work in general. It is very difficult for me to analyze your problem if you don’t try the AddOn on a blank project first.

AddOn Installation Errors

Finally, make sure to read the ReadMe’s of your freshly imported AddOns! Some AddOns require core changes, and if you don’t apply them, they will cause errors. Be sure to check the ReadMe and follow all checks before posting a error report on the Discord Server – thank you!

Errors after Official Update

Every time you update uMMORPG to the latest version, it will overwrite all changes made to your core files. In addition, the Scripting Defines will be messed up. You will have to re-apply all core changes again and clear your scripting defines, as explained above.

Errors after Switching Build Target

Clear your scripting defines to solve those errors. The Scripting Defines are unique for each one of your target builds.
