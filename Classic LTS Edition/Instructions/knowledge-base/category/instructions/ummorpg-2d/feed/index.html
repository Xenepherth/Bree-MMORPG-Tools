Description

Filters both character names and chat messages for bad words and denies them if they contain those (blocks the name from being used or the message from being sent). You can maintain a list of “bad words” that is used in both cases.

Core Changes

  1. [Required] Open the following script NetworkManagerMMO.cs. See Changes...

    Comment Out/Delete this line: 
        public bool IsAllowedCharacterName(string characterName)
    {
        // not too long?
        // only contains letters, number and underscore and not empty (+)?
        // (important for database safety etc.)
        return characterName.Length <= characterNameMaxLength &&
               Regex.IsMatch(characterName, @"^[a-zA-Z0-9_]+$");
    }
  2. [Required] Open the following script PlayerChat.cs. See Changes...

    Locate this line: 
    if (!string.IsNullOrWhiteSpace(text))

    Modify to match:

    if (UCE_IsAllowedChatText(text))

Installation

  1. [Required] Create a new “Wordfilter” list via the context menu.
  2. [Required] Add all words to the list you want to ban from both chat and character names.
  3. [Required] Select NetworkManager in your scene and add the Wordfilter object to the new slot in the inspector.
  4. [Required] Select the Chat component on each one of your player prefabs and add the Wordfilter object to the new slot in the inspector.

Screenshot

Extra

  • When a bad word is contained the character will not be created, the player gets a popup with “error”, thats all.
  • When a chat message contains a bad word, the message is not being sent. That also reduces network bandwith.

Description

This weight system adds a weight to every item in the game. Players are able to carry a maximum weight according to one Attribute. You can then specify a “burdened” buff that kicks in, when the players inventory total weight reaches a certain threshold. The burdened buff can feature several levels to simulate levels of burden and is refreshed or removed automatically every second.

Installation

  1. [Required] Check the “Burdened” buff in this AddOns Resources > Skills folder or create a new one via the context menu.
  2. [Required] Move the “Burdened” buff from this AddOns Resources > Skills folder into your projects main Resources > Skills folder.
  3. [Required] Do not keep a duplicate skill of the same name and use only one Resources folder in your project.
  4. [Optional] The “Burdened” buff can be edited and have levels. Set your penalties there.
  5. [Optional] Check your player prefabs for the new weight system settings and edit them to your liking.

Screenshot

Extra

  • Please note this system does not limit the amount of items that can be carried. It just penalizes carrying too many items.
  • Should the Burdened buff be removed from the player (via Healing/Debuff) it will be re-applied about a second later again automatically.
  • The amount of levels your skill will require, depends on the range of total weight / current weight in your game. You can even have just 1 level, but there can be a much finer granularity if you wish. The formula is quite simple: Current Weight divided by Max Weight.
  • There is a “max level” for the burdened skill that you can set, to prevent it from shooting through the roof.
  • You can set a Attribute that will increase the players carrying capability. This only works with my Attributes AddOn, not the default attributes!

]]> https://indie-mmo.net/knowledge-base/unlockable-classes-2d/feed/ 0 Timed Skill Finish 2d https://indie-mmo.net/knowledge-base/timed-skill-finish-2d/ https://indie-mmo.net/knowledge-base/timed-skill-finish-2d/#respond Mon, 28 Oct 2019 15:23:27 +0000 https://indie-mmo.net/?post_type=epkb_post_type_1&p=5532 Version: 2019.100 Requirements:uMMORPG2d, ToolsScript Define:NoneCore Changes:YesTutorial Video:No

Description

Dynamically adjust when a skill is applied: Allows you to have skills activated before the Cast Bar runs out of time. This is very useful to align animations with skills. Sometimes the skill is triggered too late while waiting for the cast animation to finish playing – this AddOn solves that problem!

Core Changes

  1. [Required] Open the following script Monster.cs. See Changes...

    Locate this line: 
    if (EventSkillFinished())
{
	... lots of code here ...
}

    Modify to match:

    #if _iMMOTIMEDSKILLFINISH
		if (UCE_StateSkillFinished())
			return "IDLE";
#else
        if (EventSkillFinished())
        {
            // finished casting. apply the skill on the target.
            FinishCastSkill(skills[currentSkill]);

            // did the target die? then clear it so that the monster doesn't
            // run towards it if the target respawned
            if (target.health == 0) target = null;

            // go back to IDLE
            lastSkill = currentSkill;
            currentSkill = -1;
            return "IDLE";
        }
#endif
  2. [Required] Open the following script Player.cs. See Changes...

    Locate this line: 
    if (EventSkillFinished())
{
	... lots of code here ...
}

    Modify to match:

    #if _iMMOTIMEDSKILLFINISH
		if (UCE_StateSkillFinished())
		return "IDLE";
#else
        if (EventSkillFinished())
        {
            // apply the skill after casting is finished
            // note: we don't check the distance again. it's more fun if players
            //       still cast the skill if the target ran a few steps away
            Skill skill = skills[currentSkill];

            // apply the skill on the target
            FinishCastSkill(skill);

            // clear current skill for now
            currentSkill = -1;

            // target-based skill and no more valid target? then clear
            // (otherwise IDLE will get an unnecessary skill request and mess
            //  with targeting)
            bool validTarget = target != null && target.health > 0;
            if (currentSkill != -1 && skills[currentSkill].cancelCastIfTargetDied && !validTarget)
                currentSkill = -1;

            // use next target if the user tried to target another while casting
            UseNextTargetIfAny();

            // go back to IDLE
            return "IDLE";
        }
#endif

Installation

  1. [Required] Apply all core changes.
  2. [Required] Check out your skill templates and adjust the “ApplySkillEarlier” section.
  3. [Required] “ApplySkillEarlier” is measured in seconds and marks the time when the skill will be applied (not the time when the skill casting is finished, this means the castbar continues to tick down).

Extra

  • Can be used for both Monsters and Players, you can also do just one of the two core changes if you dont need the other.

Description

A greatly enhanced version of the built in teleporter with a whole lot of customization options. It uses a Collider to represent the teleporter, so you can place it everywhere in your scene. The teleporter is triggered when entered by a player and requires the player to confirm the teleport via a button click on a UI window.

Installation

  1. [Required] Add a new teleporter from this AddOns “Prefabs [Drag to Scene]” folder to your scene.
  2. [Required] Edit all settings and don’t forgot to assign a teleportation target.

Extra

  • The “Simple” teleporter has no options, while the “Interactable” teleporter provides options that can limit it’s interaction (requires class, item etc.).
  • In addition to the “Simple” and “Interactable” teleporters, you can also create a “Limited” Teleporter:
  • If you want to create a teleport zone with a limited player count, you have to use the objects in the “Limited Teleport” subfolder instead.
  • For a limited zone, you will need exactly one limited zone object and at least one teleporter that acts as entrance.
  • You have to link all teleporter entrances to your limited zone to make it work. Its also a good idea to corner off the zone, so it cannot be entered by normal means.
  • If you set it up correctly, you will then have a zone that only a limited number of players can enter at once – and only through the provided teleporters.
